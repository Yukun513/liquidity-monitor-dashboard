# liquidity-monitor-dashboard
# 市场流动性监控系统 (Market Liquidity Monitor)

本项目是一个自动化的量化数据处理与可视化监控系统。其核心目标是构建一条完整的数据流水线：从外部接口抓取原始行情数据，经过剔除节假日噪音和清洗计算后，结构化归档入库（SQL），并最终生成直观的流动性监控前端面板，辅助研判市场情绪与资金动向。

## 📊 监控面板核心指标 (Core Metrics)

项目最终生成的 HTML 可视化面板采用双 Y 轴设计，直观展示两大核心流动性指标的动态博弈：

1. **杠杆资金参与度 (`margin_strength`，红线，左轴)：**
   - 衡量融资融券等杠杆资金在市场中的活跃占比。
   - 作用类似于市场的“温度计”，反映高风险偏好资金的进退情绪与攻击性。
2. **市场广度 (`跑赢基准胜率`，深蓝线，右轴)：**
   - 衡量全市场个股跑赢宽基指数的比例（范围 0 - 1）。
   - 作用类似于市场的“基本盘”，反映整体行情的赚钱效应和扩散程度。

> **应用场景：** 当杠杆资金活跃但市场广度持续走低时，往往提示少数权重股在拉升，而多数个股存在流动性衰竭风险；两者共振向上则代表良性的流动性充裕环境。

## 📁 项目架构与模块说明 (Architecture)

本项目采用模块化设计，各组件分工明确，犹如一条标准的数据加工流水线：

```text
Market_Liquidity_Monitor/
│
├── config.py                 # 🎛️ 全局配置：存放数据库连接路径、API 密钥等静态参数
├── data_sources.py           # 📥 数据源获取：负责调用接口抓取外部原始行情数据
├── utils.py                  # 🛠️ 通用工具箱：包含日期计算、格式转换等底层共用函数
├── Chinese_special_holiday.txt # 📅 静态日历：用于剔除节假日，精准校准交易日
│
├── sync_stock_daily.py       # ⚙️ 基础同步流水线：每日定时拉取全市场基础数据并入库
├── index_update.py           # ⚙️ 核心加工流水线：计算杠杆强度与胜率，清洗后写入 SQL
├── liquidity_monitor.py      # 📊 面板生成引擎：读取入库数据，生成可视化监控图表
│
├── requirements.txt          # 📦 运行环境清单：项目依赖的第三方 Python 库
├── .gitignore                # 🙈 Git 拦截规则：忽略生成的 HTML 面板文件与本地临时缓存
└── README.md                 # 📖 项目说明文档
